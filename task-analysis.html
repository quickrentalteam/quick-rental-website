Task Analysis

When it comes to apartment hunting, many hours, days weeks and even months are wasted looking for the right apartment that suits the user.

Task Analysis

At the University of the West Indies, finding the right apartment is extremely difficult and time-consuming since many apartments are not advertised, outside of walking distance and not secure. Additionally, landlords are difficult to contact and featured landlords are simply unpleasant . Students must take great care and effort to locate an apartment that fits within their budget and level of comfort.

01

Creating an account

Goal

The user can create a student or landlord account on the mobile application. This account will be used for identification and authentication of the user (e.g. validating their email) and may be attached to any maintenance/service requests that the user makes.

Preconditions

  1. User has required hardware (phone, desktop, laptop, tablet).
  2. User has successfully installed the software and has opened the app.

Exceptions

  1. User attempts to use an email address already associated with an account. This shouldn’t occur very often.
  2. User enters an invalid or malformed email address.
  3. User specifies an inadequate password.
  4. User attempts to specify no password.


This task should occur once. Upon account creation, we must ensure that the user does not require the creation of a second account. This task should take no more than 5 minutes.

02

Login | Logout

Goal

The user can login with their username/email and password or logout.

Preconditions

  1. The user has opened the app.
  2. The user already has an account created.

Exceptions

  1. The user forgot their password, which they can recover by clicking on ‘forgot password’.
  2. The user enters the wrong username/email or an email that is not registered with the app.
  3. User attempts to enter no password.
  4. User attempts to specify no password.


This task will take less than 1 minute and will allow the user to access all features of the application.

03

Search Apartments Using Filters

Goal

The user can browse for apartments by filtering their data. The filters include, but are not limited to:

  • Location

    A map of UWI and the surrounding areas consisting of the location of the apartment is shown. The distance in km at walking distance will also be provided. Thus, you can select e.g. short distance to UWI as a filter. The user can find apartments using specific location through GPS.

  • Price Range

    The apartments at a certain price range from low to high will be shown.

  • Ratings

    From previous ratings of the apartments by students, the apartments with the highest to lowest rating will be shown.

  • Availability

    The apartments that are available will be shown.

  • Unfurnished | Furnished

    The apartments that are unfurnished or furnished will be shown.

  • Studio | Shared

    The apartments with single or double occupancy will be shown.

  • A checklist for if the apartment has utilities include, air-conditioning, security, parking or pets allowed can be chosen from.

  • The visitor’s policy, either restricted, restricted after a certain time or no restrictions will be options.

  • If the apartment is coed or same-sex can be chosen from as well.

Preconditions

  1. The user must already have an account and is logged in.

Subtasks

  1. The user will select the filters mentioned above before clicking the ‘Browse’ button e.g. the user will choose from a range of prices, tick all the boxes of features they want to find in an apartment like studio, furnished, available etc. and then click on the button 'Browse'. A results page will be shown with all apartments that match the selected criteria.

Exceptions

  1. User can get no results upon searching for apartment with certain criteria.


It will not take more than 3 minutes to choose all the criteria the user wants in an apartment. The user will be able to easily find what they want, and if there are no results, a suggestion will be made to an apartment that has most of their criteria selected.

04

Add | Update | Delete Apartment Information by the Landlord

Includes upload photos, location, prices, availability etc.

Goal

  • ADD
    The landlord can create a listing for rental accommodation. They can upload photos and videos of the interior and exterior with their listing as well as publish prices, availability, facilities and other criteria of the apartment.
  • UPDATE
    The landlord can update any information of the apartment based on room availability.
  • DELETE
    The landlord can also delete their apartment listing is they desire to.

    • Preconditions

    1. The user must already have an account and is logged in.
    2. The user is a landlord.

    Subtasks

    1. The landlord can upload photos/videos of the interior and exterior of the apartment by clicking on the 'Upload' button for photos and videos respectively.
    2. The landlord must add the price, location, security, facilities included, if it is shared, a studio, furnished, unfurnished, allows pets, has parking, air-conditioning, coed, same-sex etc. A form will be given to the landlord after account creation, where the user can easily enter these data.

    Exceptions

    1. The user can enter false information - We will require the trust of the landlord to upload the accurate photos/videos of the apartment and to be honest. If it is found that the information was false, the user will be reported.


    This task will take no longer than 10 minutes (depending on when the user takes the photos and videos of the apartment). If there are already video/photos, the task will take 5 minutes or less. The landlords will get more renters this way depending on the usage of the app.

05

Contact Landlord Through Messaging Platform

Goal

The students can contact the landlord via the email or phone number entered by the landlord. However, there will be an option to instant message the landlord on a messenger platform, upon urgency or for a quicker response depending on if the landlord is online.

Preconditions

  1. The user must already have an account and is logged in.
  2. Both the landlord and student must be online.
  3. Once you are on the details page of an apartment, you can message the landlord directly, where messages will have a reference to the apartment since landlords may have several listings.
  4. General enquiries to the landlord will be done via clicking on the landlord's avatar.

Exceptions

  1. The student can abuse the messaging platform and send nonsensical messages to the landlord. If this occurs, the user will be reported.


This task will take as long as the conversation between the landlord and student. The feature is important as many students do not get into contact with landlords in a timely manner or the number provided may be outdated.

06

Leaving a Review

Goal

Previous students can leave reviews of apartments they stayed at and give an overall rating, by clicking the 'Review' button.

Preconditions

  1. The user must already have an account and is logged in.
  2. The user must have stayed at the apartment. This will be validated to ensure no one randomly rates an apartment.
  3. The user must be on the specific apartment page.

Subtasks

  1. The user can choose the amount of stars out of five to rate the apartment.
  2. The user can upload photos of the apartment to be included in the review.

Exceptions

  1. The user can enter a bad rating without having stayed at the apartment. This will be accounted for as we will validate that the user stayed at the apartment.


This task will take about than three minutes. A user can choose the amount of stars out of five to rate the apartment, as well as include photos.

07

Browse

Goal

The user can browse through the home page or browse through the results after searching for apartments with specific criteria.

Preconditions

  1. The user must already have an account and is logged in.

Subtasks

  1. The user can view a listing of apartments on the home page. This task is the 'Apartment Listing' task, task 09.
  2. The user can perform the 'Filter' task from task 03 above, and browse through the results.
  3. When the user finds an apartment they like, the would click on that apartment and view the apartment details. The apartment details task is described on task 08.



This length of time for this task will be based on how many apartments the user browses through. This is the main functionality of QuickRental as the user will find apartments they like and get to see all details about it.

08

Apartments Details

Goal

Users can view the details of the apartments, as well as tour the apartment virtually, view the map for the apartment and view reviews left for the apartment.

Preconditions

  1. The user must already have an account and is logged in.
  2. The user must be on the specific apartment page.

Subtasks

Viewing Reviews
  1. By clicking the 'Reviews' button, the user can read see the ratings, as well as comments for each review.
  2. On the reviews section, the user can read the comments on each review and thumbs up or down the review and comments if they wish.
Virtual Apartment Tour
  1. By clicking the 'Tour' button, the user is taken to the virtual tour page where they can view the instructions and the tour.
  2. The user interactively and virtually walk through the apartment and view different rooms.
Viewing Maps
  1. By clicking the 'Map' button, the user is taken to the map of the apartment, which uses google maps.
  2. The user can zoom on the maps to see the streets and amenities near the apartment, as well as view the maps on fullscreen.
  3. The user can see the walking and driving distance to U.W.I, in minutes
Contact the Landlord
  1. Contacting the landlord is also a main task as seen in task 05

Exceptions

Virtual Apartment Tour
  1. The photos/videos uploaded by the landlord may be too vague or purposefully exclude pertinent information.


This task will take about 10 minutes or more. These ratings are important and will be a filter for the student to choose from, when browsing for apartments. For the virtual tours, users will be able to see the apartment before going to visit. This will help them narrow down their search and time as they will not have to drive around and check every available apartment. For the maps, there will be the walking and driving distance to U.W.I, where the user can zoom anywhere on the map due to google maps being used. If they wish to, they can view the maps on full screen.

09

Apartment Listing

Goal

The user can view a list of all the apartments based on their previous query or on the home page.

Preconditions

  1. The user must already have an account and is logged in.

Exceptions

  1. The student can abuse the messaging platform and send nonsensical messages to the landlord. If this occurs, the user will be reported.


This task take around two minutes. The listing of apartments can be from the home screen or after entering filters on the filter screen which produces a list of apartments as results.

10

Favourite | Bookmark an Apartment

Goal

The student can bookmark all of their favourite apartments they have viewed so far, to look back at a later time or to choose the best from those selected apartments.

Preconditions

  1. The user must already have an account and is logged in.
  2. The user must have already viewed at least one apartment.
  3. The user must be on the specific apartment page they wish to bookmark or can bookmark right on the results and home page where apartments are displayed.

Exceptions

  1. An apartment you bookmark may become unavailable. To account for this, the user will be notified of this event.


This is a main part of the functionality of the app, so it will be a responsive feature; not taking more than a second to bookmark with a single click on a 'heart' icon. It is important for the user to be able to see a list of their bookmarked apartments and, therefore, narrow their search.

11

Notification

Goal

The user can receive notifications concerning apartments they have bookmarked, when it becomes unavailable or changes are made to the details.

Preconditions

  1. The user must already have an account and is logged in.
  2. The user must have already bookmarked at least one apartment.



This is an important functionality of QuickRental as knowing when one of your favourite apartment becomes available instantly will help the user to be successful in renting that apartment before it becomes unavailable again.
